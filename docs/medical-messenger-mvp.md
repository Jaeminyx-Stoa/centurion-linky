# 통합 미용의료 AI 상담 플랫폼 - MVP 기획서

> **프로젝트 목적**: 외국인 대상 미용의료 서비스를 제공하는 클리닉(예: 데이뷰의원)을 위한
> 멀티 메신저 통합 AI 상담 플랫폼. 다양한 메신저로 들어오는 외국인 고객 문의를
> AI가 자동으로 응대하고, 예약 전환까지 이끄는 것이 핵심.

> **핵심 철학**: 사람은 "파수꾼(수퍼바이저)" 역할만 한다. 시니어급 한 명이
> AI를 감독하는 구조. 일상적 상담은 AI가 전부 처리한다.

## 1. 멀티 메신저 통합

<!--
  [기획 의도]
  미용의료 외국인 환자 유치에서 가장 큰 문제는 나라마다 쓰는 메신저가 다르다는 것.
  일본은 LINE, 중국은 WeChat, 서양은 Instagram/WhatsApp 등.
  클리닉 직원이 앱을 왔다갔다 하면서 응대하면 누락이 생기고, 응답이 느려지고,
  결국 고객을 놓친다. → 하나의 화면에서 전부 처리해야 한다.
-->

### 1-1. 지원 메신저 (6개)

| 메신저 | API | 주요 타겟 국가 |
|--------|-----|---------------|
| Instagram DM | Meta Business Platform | 글로벌 (특히 일본, 미국, 동남아) |
| LINE | LINE Messaging API | 일본, 대만, 태국 |
| Facebook Messenger | Meta Business Platform | 글로벌 (특히 동남아, 미국) |
| WhatsApp | Meta Business Platform | 미국, 유럽, 동남아 |
| KakaoTalk | 카카오톡 채널 API | 한국 |
| Telegram | Telegram Bot API | 러시아, 중동, 유럽 |

<!--
  [기획 의도]
  Meta 플랫폼 하나로 Instagram + Facebook + WhatsApp 3개를 동시에 커버.
  LINE은 일본/대만 고객에 필수. Telegram은 가장 API가 쉬워서 개발/테스트용으로도 좋음.
  카카오톡은 한국 고객 또는 한국 내 외국인 대응용.
-->

### 1-2. 멀티 계정 지원

<!--
  [기획 의도]
  데이뷰의원처럼 하나의 클리닉이 나라별로 여러 계정을 운영하는 경우가 많다.
  daybeauclinic_tw (대만), daybeauclinic_eng (영어권), daybeauclinic_jp (일본) 등.
  이 계정들이 로그인 자체가 다를 수 있음 (계정 전환이 아니라 별도 로그인).
  → 각 계정마다 별도 API 키를 등록하는 구조로 해결.
-->

```
예시: 데이뷰의원의 계정들

Instagram:
├── daybeauclinic_tw      (대만용)
├── daybeauclinic_eng     (영어권용)
├── daybeauclinic_jp      (일본용)
└── daybeauclinic_global  (글로벌)

LINE:
├── 데이뷰 公式 (일본)
└── 데이뷰 台灣 (대만)

KakaoTalk:
└── 데이뷰의원 채널

→ 전부 하나의 대시보드로 통합
→ 계정 수 제한 없음
```

### 1-3. 메시지 태깅

<!--
  [기획 의도]
  모든 메시지가 한 곳에 모이면, "이게 어디서 온 건지" 헷갈릴 수 있다.
  모든 메시지에 [메신저 종류] + [어떤 계정]을 태그로 표기해야
  직원이 맥락을 바로 파악할 수 있다.
-->

- 모든 메시지에 `[메신저]` + `[계정]` 태그 자동 표기
- 메신저별 / 계정별 / 상태별 필터링 지원

## 2. AI 답변 엔진 (3레이어 분리 구조)

<!--
  [기획 의도 - 핵심 중의 핵심]

  "말하는 법"과 "말하는 내용"을 반드시 분리해야 한다.

  이유:
  1. 클리닉마다 시술 정보가 다르다 (의사의 전문성, 고집, 매출 전략이 다 다름)
     → 보톡스 효과: A클리닉 3개월, B클리닉 2개월
     → 최소 간격: A클리닉 2개월, B클리닉 1개월
     → 이건 "교과서적 정답"이 아니라 "클리닉별 방침"이다.

  2. 기존 CS 상담사들은 100% 확률로 부정확한 의학 지식을 가지고 있다.
     → CS 데이터에서 의료 정보를 학습하면 잘못된 정보를 퍼뜨리게 된다.
     → CS 데이터에서는 "상담 스킬"만 추출해야 한다.

  3. 말투/어조는 CS 데이터보다 인터넷 SNS/커뮤니티가 더 좋은 소스다.
     → CS 데이터: 수천 건, 상담 톤에 한정
     → SNS 데이터: 수억 건, 진짜 사람들의 자연스러운 말투
     → 나라별로 해당 나라 SNS를 쓰면 가장 자연스러운 표현을 얻을 수 있다.
-->

### 레이어 ① 지식 레이어 — "뭘 말할지"

```
우선순위:

1순위: 클리닉 자체 매뉴얼 (의사가 직접 세팅)
       → "우리 클리닉은 보톡스 간격 최소 2개월"
       → "우리는 필러 지속기간 12개월로 안내"

2순위: 의학 교과서 / 검증된 의료 정보
       → 기본값으로 깔려있음
       → 클리닉 매뉴얼에 없는 항목은 이걸로 답변

❌ CS 상담 데이터에서 의료 정보는 절대 가져오지 않음
```

<!--
  [기획 의도]
  클리닉 매뉴얼이 1순위인 이유: 의사 개개인이 전문가이고,
  각자의 방침이 있다. 이걸 무시하고 교과서만 따르면 클리닉과 충돌한다.
  교과서는 "기본값"일 뿐이고, 클리닉 설정이 있으면 반드시 그걸 따른다.
-->

### 레이어 ② 말투 레이어 — "어떻게 말할지"

```
소스: 각 나라 SNS / 커뮤니티 데이터

일본: Twitter(X), LINE 커뮤니티
중국: 小红书(샤오홍슈), WeChat 커뮤니티
한국: 네이버 카페, 인스타 댓글
미국: Instagram, Reddit
```

<!--
  [기획 의도]
  처음에는 CS 상담 데이터로 말투를 학습하려 했으나,
  "말하는 법"을 분리하면 SNS/커뮤니티 데이터가 훨씬 우월하다는 결론.
  - 데이터 양: 수천 건 vs 수억 건
  - 자연스러움: 상담 말투 한정 vs 실제 사람들의 다양한 표현
  - 나라별 커버: 특정 클리닉 고객만 vs 각 나라 전체
  - 최신 트렌드: 느림 vs 실시간 반영
-->

### 레이어 ③ 상담 스킬 레이어 — "어떻게 설득할지"

```
소스: 기존 CS 상담 데이터

✅ 가져오는 것 (상담 스킬만):
├── 고객이 망설일 때 어떻게 설득하는지
├── 불안해하는 고객을 어떻게 안심시키는지
├── 예약으로 어떻게 자연스럽게 유도하는지
└── 가격 질문에 어떻게 응대하는지

❌ 가져오지 않는 것:
├── 의료 정보 (부정확할 수 있음) → 지식 레이어가 담당
└── 말투/어조 → SNS 레이어가 담당
```

<!--
  [기획 의도]
  CS 데이터의 진짜 가치는 "세일즈 기술"과 "예약 전환 유도 기술"이다.
  상담사들이 수년간 쌓은 노하우 중 가장 값진 부분.
  특히 어떨 때 고객이 긍정적으로 반응하는지,
  어떤 화법이 예약으로 이어지는지를 AI가 분석해서 활용한다.
-->

### 답변 생성 흐름 예시

```
고객 (일본인): "ボトックスはどのくらい持ちますか？"
              (보톡스 얼마나 지속되나요?)

① 지식 레이어: 데이뷰의원 설정 → "보톡스 3개월"
② 말투 레이어: 일본 SNS 기반 → 자연스러운 일본어
③ 상담 스킬: 정보 제공 후 예약 유도 패턴 적용

→ "ボトックスの効果は約3ヶ月ほど持続いたします✨
    個人差はございますが、定期的に施術されますと
    より長く効果を実感いただけますよ😊"
```

## 4. 자동 번역

<!--
  [기획 의도]
  단순 번역이 아니라 "미용의료 전문 번역"이 되어야 한다.
  "물광주사"를 직역하면 말이 안 되고, "쁘띠성형"은 한국 특유의 표현이다.
  → 전문 용어 사전을 먼저 거치고, 나머지를 AI가 번역하는 2단계 구조.
-->

### 4-1. 미용의료 전문 용어 사전

```
용어 사전 예시:

물광주사:
  영어: Skin Booster Injection
  일본어: 水光注射
  중국어: 水光注射

쁘띠성형:
  영어: Non-surgical Procedure
  일본어: プチ整形
  중국어: 微整形

→ 클리닉이 직접 추가/수정 가능
→ 새로운 시술 나오면 사전에 추가만 하면 됨
```

### 4-2. 번역 흐름

```
수신 (외국어 → 한국어):
  고객 메시지 → 용어 사전 매칭 → AI 번역 → 직원이 한국어로 읽음

발신 (한국어 → 외국어):
  직원/AI 답변 → 용어 사전 매칭 → AI 번역 → 고객 언어로 자동 전송
```

## 5. 자동 답변 시스템

<!--
  [기획 의도]
  상담의 99%는 정보성 질문이다. 가격, 위치, 시술 설명, 예약 방법 등.
  이걸 사람이 일일이 답하는 건 비효율적이다.
  AI가 답변 라이브러리와 지식 레이어를 기반으로 자동 처리하고,
  사람은 진짜 위급한 상황(컴플레인, 부작용)에만 개입한다.

  핵심: "AI가 할 수 있는 건 AI가 하고, 사람만 할 수 있는 건 사람이 한다."
-->

### 5-1. 자동 답변 (99% 메시지)

```
AI가 자동 처리하는 것:
├── 가격 문의
├── 시술 정보 문의
├── 위치 / 오시는 길
├── 운영 시간
├── 예약 방법 / 예약 진행
├── 시술 후 주의사항
├── 후기 / 리뷰 관련
├── 주차 안내
└── 기타 정보성 질문
```

### 5-2. 사람 연결 (1% 메시지)

```
사람이 개입해야 하는 것:
├── 컴플레인 (불만 접수)
├── 부작용 보고 ("수술 후 얼굴이 부었어요")
├── 환불 요청
├── 의료사고 관련
└── 기타 위급/민감한 상황

감지 방식: AI가 키워드 + 문맥 분석
  → "부작용", "아파요", "피가", "환불", "신고", "불만" 등
  → 담당자에게 즉시 알림 (카톡/문자/이메일)
  → 고객에게 "담당자가 곧 연락드리겠습니다" 자동 전송
```

### 5-3. 답변 라이브러리

<!--
  [기획 의도]
  AI가 매번 새로 답변을 생성하는 것보다,
  검증된 답변 라이브러리를 기반으로 하는 게 정확도가 높다.
  라이브러리는 클리닉이 직접 관리할 수 있어야 한다.
-->

```
📁 답변 라이브러리
├── 💰 가격 문의
│   ├── 보톡스 가격 안내
│   ├── 필러 가격 안내
│   └── 리프팅 가격 안내
├── 📅 예약
│   ├── 예약 방법 안내
│   ├── 예약 변경/취소
│   └── 상담 가능 시간
├── 📍 위치/오시는 길
├── 💉 시술 정보
│   ├── 보톡스 설명
│   ├── 필러 설명
│   └── 시술 후 주의사항
└── 🌐 외국인 전용
    ├── 비자/체류 관련
    └── 공항 픽업 안내
```

## 6. 사람다움 (자연스러운 AI)

<!--
  [기획 의도]
  고객이 AI와 대화하고 있다는 느낌을 최소화해야 한다.
  핵심은 "사람처럼 보이는 것"이 아니라 "사람만큼 자연스러운 것".
  법적으로 AI 표기는 의무이므로 프로필에 고지하되,
  대화 품질 자체는 사람과 구분이 안 될 정도로 만든다.
-->

### 6-1. 자연스러움 기법

| 기법 | 설명 |
|------|------|
| 답변 딜레이 | 1~3초 딜레이 (사람이 타이핑하는 느낌) |
| 입력 중 표시 | 메신저의 "입력 중..." 상태 표시 활용 |
| 시간대별 인사 | 아침: "좋은 아침이에요~" / 밤: "늦은 시간에 감사해요!" |
| 대화 기억 | "아까 보톡스 물어보셨잖아요~" 처럼 이전 대화 참고 |
| 페르소나 설정 | 클리닉별 상담사 캐릭터 (이름, 성격, 말투) 설정 가능 |

### 6-2. 법적 요건

```
프로필에 표기:
"AI 상담사가 도와드리고 있으며, 필요시 전문 상담사가 연결됩니다"

→ 법적으로 안전
→ 고객도 크게 신경 안 씀 (대화가 자연스러우면)
```

## 7. 고객 만족도 실시간 측정

<!--
  [기획 의도]
  사람이 모든 대화를 일일이 읽을 수 없다. 그래서 AI가 실시간으로
  고객 만족도를 측정하고, 위험한 대화만 사람에게 알려준다.
  사람은 "파수꾼" 역할만 하면 된다. 시니어급 한 명으로 충분.

  핵심 질문: "AI가 만족도를 정확히 측정할 수 있는가?"
  → 100% 정확하진 않지만, 사람보다 빠르고 일관적.
  → 처음엔 70~80% 정확도, 수퍼바이저 피드백으로 빠르게 향상.
-->

### 7-1. 분석 신호

```
📊 언어 신호 (가장 정확)
├── 부정 단어: "아니요", "됐어요", "그만", "싫어요"
├── 불만 표현: "아까도 말했는데", "왜 자꾸", "답답"
├── 긍정 표현: "오 좋아요", "감사해요", "궁금한데"
└── 욕설 / 비꼬기 감지

📊 행동 신호 (꽤 정확)
├── 답장 속도 느려짐 → 관심 떨어짐
├── 메시지 길이 줄어듦 → "네", "ㅇㅇ" → 냉담해짐
├── 같은 질문 반복 → 답변에 불만족
├── 이모티콘 사라짐 → 감정 변화
└── 읽고 답 안 함 → 이탈 위험

📊 대화 흐름 신호 (보통)
├── 예약 쪽으로 이동 → 만족도 상승
├── 가격만 계속 물음 → 망설임
├── 다른 병원 언급 → 이탈 위험
└── "생각해볼게요" → 사실상 거절
```

### 7-2. 경고 체계

```
90~100점 🟢  완벽한 상담 진행 중
70~89점  🟡  정상, AI 자동 답변 계속
50~69점  🟠  ⚠️ 수퍼바이저에게 모니터링 알림
               → AI는 계속 답변 (멈추지 않음)
               → 사람은 지켜보기만
50점 미만 🔴  🚨 수퍼바이저에게 개입 알림
               → "직접 대응 권장합니다"
               → AI는 톤을 보수적으로 전환
               → 사람이 바로 이어받을 수 있음
```

### 7-3. 정확도 향상

<!--
  [기획 의도]
  초기에 AI 판단이 틀릴 수 있다. 이때 수퍼바이저가
  "이건 80점이야, 이 고객은 원래 말이 짧아"처럼 교정해주면
  AI가 이 피드백을 학습해서 점점 정확해진다.
  수퍼바이저가 AI를 훈련시키는 구조.
-->

```
초기: AI 판단 + 수퍼바이저 교정
  AI: "이 고객 65점입니다"
  수퍼바이저: "아니 이건 80점이야, 이 고객은 원래 말이 짧아"
  → AI가 피드백 학습 → 점점 정확해짐
  → 3개월 후: 사람과 거의 같은 판단력
```

## 8. AI 세일즈 스킬 자동 향상

<!--
  [기획 의도]
  AI가 단순히 정보를 전달하는 게 아니라, "예약 전환"을 이끌어야 한다.
  이것이 이 플랫폼의 진짜 가치.
  AI가 모든 상담 데이터를 분석해서 "어떤 화법이 예약으로 이어지는지"를
  통계적으로 찾아내고, 그 패턴을 점점 더 많이 사용한다.
  사람 상담사 100명의 경험을 합친 것보다 빠르게 최적의 세일즈 화법을 찾아낸다.
-->

### 8-1. 자동 학습 루프

```
AI 답변 전송
    ↓
결과 추적
├── 예약으로 이어졌나?  → +10점
├── 고객이 긍정 반응?  → +5점
├── 대화가 계속됐나?   → +3점
└── 이탈했나?         → -5점
    ↓
높은 점수 패턴을 더 자주 사용
    ↓
반복 (점점 전환율 상승)
```

### 8-2. 성공/실패 패턴 분석 예시

```
❌ 실패 패턴 (예약 전환율 12%)
  고객: "보톡스 얼마예요?"
  AI: "50만원입니다"
  고객: "좀 비싸네요..."
  AI: "저희가 실력이 좋아서요"
  고객: (읽씹)

✅ 성공 패턴 (예약 전환율 73%)
  고객: "보톡스 얼마예요?"
  AI: "어떤 부위 생각하고 계세요?😊"
  고객: "이마랑 미간이요"
  AI: "이마+미간 같이 하시면 15만원이에요!
       지금 2월 첫방문 혜택도 있어서
       상담만 먼저 받아보실래요?✨"
  고객: "오 네 언제 가능해요?"
```

### 8-3. A/B 테스트

<!--
  [기획 의도]
  같은 질문에 대해 여러 화법을 동시에 테스트하고,
  전환율이 높은 화법의 비중을 자동으로 높인다.
  사람은 이런 실험을 할 수 없지만, AI는 수천 건을 동시에 돌릴 수 있다.
-->

```
같은 질문 "보톡스 얼마예요?"에 대해:

A안: 가격 먼저 → 상담 유도
B안: 부위 질문 → 맞춤 가격 → 예약 유도

→ 1000건 돌려보고 B안이 전환율 높으면
→ 자동으로 B안 비중을 높임
```

## 9. AI vs AI 시뮬레이션 훈련

<!--
  [기획 의도]
  실제 고객으로 A/B 테스트하면 데이터 모으는 데 몇 달 걸린다.
  AI 고객 페르소나를 만들어서 AI끼리 시뮬레이션하면
  하루에 수천 건을 테스트할 수 있다.

  단, AI 시뮬레이션만으로는 한계가 있다.
  AI 고객은 "너무 합리적"이라 실제 사람의 비합리적 행동을 못 따라한다.
  → AI 시뮬레이션으로 빠르게 1차 필터링 → 실제 데이터로 2차 검증/보정

  이러면 출시 전에도 이미 어느 정도 학습된 상태로 시작할 수 있다.
-->

### 9-1. AI 고객 페르소나

```
👩 일본인 30대 여성 "유코"
├── 성격: 신중, 질문 많음, 가격 민감
├── 특징: 한국 미용 처음, 불안함 큼
└── 패턴: 3번 이상 확인해야 예약

👨 중국인 40대 남성 "웨이"
├── 성격: 직접적, 결과 중심
├── 특징: VIP 대우 기대, 가격보다 실력
└── 패턴: 비포/애프터 요구, 빠른 결정

👩 미국인 20대 여성 "제시카"
├── 성격: 캐주얼, 트렌드 민감
├── 특징: 인스타에서 봤다며 문의
└── 패턴: 후기/리뷰 중시

👧 베트남인 20대 여성 "린"
├── 성격: 친근, 할인 관심
├── 특징: 친구랑 같이 오고 싶어함
└── 패턴: 그룹 할인, 패키지 관심
```

### 9-2. 시뮬레이션 프로세스

```
매일 밤 자동 실행:

상담 AI ←→ 고객 AI (유코)    × 500회
상담 AI ←→ 고객 AI (웨이)    × 500회
상담 AI ←→ 고객 AI (제시카)  × 500회
상담 AI ←→ 고객 AI (린)      × 500회
           │
           ▼
      결과 분석
      "유코 타입에게는 B화법이 전환율 높음"
      "웨이 타입에게는 A화법이 전환율 높음"
           │
           ▼
      다음날 실제 고객에게 적용
           │
           ▼
      실제 결과로 AI 고객 페르소나 보정
      "유코 타입이 실제보다 너무 쉽게 전환됨 → 더 까다롭게 조정"
```

### 9-3. 정확도

| 단계 | 방법 | 속도 | 정확도 |
|------|------|:----:|:------:|
| 1차 필터 | AI vs AI 시뮬레이션 | 하루에 만 건 | 70~80% |
| 2차 검증 | 실제 고객 데이터 | 천천히 쌓임 | 95%+ |

## 11. 결제 시스템

<!--
  [기획 의도]
  모든 결제를 QR 결제(결제 링크)로 통일한다. 원내 카드단말기 사용 안 함.

  이유:
  1. CRM을 위해서. 모든 결제가 우리 시스템을 통과해야
     결제 후 자동 프로세스(만족도 조사, 리뷰 요청, 재방문 리마인더)를
     빠짐없이 실행할 수 있다.
  2. 카드단말기로 결제하면 우리 시스템이 결제 완료를 감지할 수 없어서
     CRM 자동화가 끊긴다.
  3. 외국인 카드가 한국 단말기에서 안 되는 문제도 해결.

  결제 파트너 전략:
  - 킹오더브라더스 (국내 PG 파트너) → 메인. 국내+해외 결제수단 커버.
  - 킹오더브라더스로 안 되는 것(Alipay 등) → 해당 결제수단 전문 파트너와 개별 계약.
    예: 알리엑스 (Alipay 전문)
  - Stripe → 위 파트너들로 커버 안 되는 나라 백업용.
  - 향후 새로운 결제수단이 필요할 때마다 그에 맞는 파트너를 추가 계약하는 유연한 구조.
-->

### 11-1. 결제 전략

```
⚠️ 핵심: 모든 결제를 QR 결제(결제 링크)로 통일

예약금 (원격): 메신저로 QR/결제 링크 발송 → 고객 폰에서 결제
잔금 (원내):   메신저로 QR/결제 링크 발송 → 고객 폰에서 결제
추가 결제:     메신저로 QR/결제 링크 발송 → 고객 폰에서 결제

→ 원내 카드단말기 사용하지 않음
→ 모든 결제가 우리 시스템을 통과 → CRM 자동화의 핵심 전제 조건
```

### 11-2. 결제 파트너 구조

<!--
  [기획 의도]
  킹오더브라더스를 메인 PG로 두되, 킹오더브라더스가 커버 못 하는
  결제수단은 해당 분야 전문 파트너와 개별 계약한다.
  "하나의 PG에 올인"이 아니라 "결제수단별 최적 파트너" 전략.
  새로운 시장(나라)이 추가될 때마다 유연하게 파트너를 붙일 수 있다.
-->

```
┌─────────────────────────────────────────────┐
│  결제 파트너 구조                              │
│                                             │
│  [메인] 킹오더브라더스 (국내 PG 파트너)         │
│  ├── 한국: 카카오페이, 네이버페이, 카드 등       │
│  ├── 일본: LINE Pay, 카드                     │
│  ├── 대만: LINE Pay, 카드                     │
│  └── 기타 킹오더가 지원하는 결제수단            │
│                                             │
│  [개별 계약] 결제수단별 전문 파트너              │
│  ├── 알리엑스 → Alipay (중국 고객)             │
│  ├── (필요시) WeChat Pay 전문 파트너            │
│  ├── (필요시) GrabPay 등 동남아 전문 파트너     │
│  └── 시장 확장 시 그때그때 맞는 파트너 추가      │
│                                             │
│  [백업] Stripe                               │
│  └── 위 파트너들로 안 되는 나라/결제수단 커버    │
└─────────────────────────────────────────────┘
```

### 11-3. 나라별 최적 결제수단

| 나라 | 결제수단 | 파트너 | 비고 |
|------|---------|--------|------|
| 🇨🇳 중국 | Alipay / WeChat Pay | 알리엑스 (+ 추가 파트너) | 킹오더로 안 되는 경우 개별 계약 |
| 🇯🇵 일본 | LINE Pay / 카드 / 은행 송금 | 킹오더브라더스 | |
| 🇹🇼 대만 | LINE Pay / 카드 | 킹오더브라더스 | |
| 🇰🇷 한국 | 카카오페이 / 네이버페이 / 카드 | 킹오더브라더스 | |
| 🇺🇸 미국/유럽 | 카드 / Apple Pay | Stripe (백업) | |
| 기타 | 해당 나라 결제수단 | 그때그때 파트너 계약 | 유연하게 확장 |

<!--
  [기획 의도]
  PG의 PG(결제 대행의 대행)가 되어 수수료 차익을 먹는 모델도 검토했으나,
  전자결제대행업 라이선스(자본금 10억+, 금감원 인가)가 필요해서
  MVP 단계에서는 현실적이지 않음.
  → MVP: 클리닉이 직접 결제 받고, 우리는 유치 수수료(10%)에 집중.

## 12. CRM (고객 관계 관리)

<!--
  [기획 의도]
  이것이 모든 결제를 QR로 통일하는 핵심 이유.
  결제가 우리 시스템을 통과해야 결제 완료 시점을 감지할 수 있고,
  그래야 아래의 모든 CRM 자동화가 빠짐없이 실행된다.
  카드단말기를 쓰면 결제 완료를 감지할 수 없어서 CRM이 끊긴다.

  CRM의 핵심 목표:
  1. 시술 후 만족도 파악 (직후 / 7일 / 14일 - 시간에 따라 만족도가 변할 수 있음)
  2. 재방문 의사 확인 → 재방문율 향상
  3. 주변 추천 의향 확인 → 구전 마케팅 (NPS 지표)
  4. 리뷰 확보 → 리뷰는 시술 직후가 만족도 가장 높을 때이므로 직후에 요청
  5. 재시술 리마인더 → 자연스러운 재방문 유도
-->

### 12-1. CRM 자동 프로세스 타임라인

```
결제 완료 (시술 완료)
│
├→ 즉시: 영수증 자동 발송
│
├→ 즉시~1시간: 리뷰 요청 ⭐
│  → "오늘 시술 어떠셨어요? 후기 남겨주시면 감사하겠습니다😊"
│  → 리뷰 링크 (Google, 네이버, Instagram 등)
│  → ⚠️ 직후가 만족도 가장 높은 시점이므로 이때 요청하는 게 핵심
│
├→ 3시간 후: 시술 후 주의사항 자동 발송
│
├→ 직후 (당일): 만족도 조사 1차 📊
│  → "오늘 시술 만족하셨나요?" (1~5점)
│  → 간단한 이모지 선택 또는 점수
│
├→ 7일 후: 만족도 조사 2차 📊
│  → "시술 후 일주일, 경과는 어떠세요?"
│  → 만족도 점수 + 부작용/불편사항 확인
│  → 재방문 의사 질문: "다음에도 방문하실 의향이 있으세요?"
│
├→ 14일 후: 만족도 조사 3차 📊
│  → "시술 후 2주, 결과에 만족하시나요?"
│  → 최종 만족도 점수
│  → 주변 추천 의향: "지인에게 추천하실 의향이 있으세요?" (NPS)
│  → 추천 의향 높으면 → 추천 코드/링크 자동 제공
│
└→ 2~3개월 후: 재시술 리마인더
   → "유코님, 보톡스 맞으신 지 3개월 되었어요~
      다음 시술 예약 도와드릴까요?😊"
   → 예약 바로 연결
```

<!--
  [기획 의도 - 만족도 조사가 3번인 이유]
  시술 직후: 시술 경험 자체에 대한 만족도 (서비스, 통증, 분위기 등)

## 13. 프로시져 허브

<!--
  [기획 의도 - 이 플랫폼의 두뇌]

  프로시져 허브는 시술(프로시져)에 대한 "모든 정보"가 모이는 곳이다.
  클리닉의 "메뉴판"이자 AI의 "지식 데이터베이스"이자 "세일즈 전략 엔진"이다.

  3가지 역할:
  1. 시술 정보 라이브러리 → AI가 고객에게 답변할 때 참조하는 의학 정보
  2. 수가(가격) 관리 → 클리닉별 가격, 이벤트, 할인 관리
  3. 세일즈 퍼포먼스 엔진 → 어떤 시술을 밀어야 하는지 점수화

  기본값은 교과서 기준으로 우리(서비스 제공자)가 세팅해놓고,
  클리닉이 자기 사정에 맞게 커스터마이징(클리닉 내부룰 우선주의)한다.

  ⚠️ 매우 중요: 프로시져 허브의 정보 중 고객에게 보여줄 것과
  보여주면 안 되는 것을 엄격히 구분해야 한다.
-->

---

### Part A. 시술 정보 라이브러리

<!--
  [기획 의도]
  AI가 고객 상담할 때 참조하는 의학/시술 정보의 원천.
  "교과서 기본값"을 우리가 꼼꼼하게 세팅해놓고,
  클리닉이 자기 방침에 맞게 덮어쓸 수 있다.
  (예: 다운타임 교과서는 3일이지만, 우리 클리닉은 5일로 안내)

  이 정보는 AI 답변의 정확성을 좌우하므로 매우 중요하다.
-->

#### 13-1. 시술 정보 항목 (시술당 입력)

```
📋 시술 기본 정보
├── 시술명 (한국어 + 영어 + 각 언어)
├── 시술 카테고리 (보톡스/필러/리프팅/레이저/수술 등)
├── 시술 설명 (이 시술이 무엇인지)
└── 효능효과 (어떤 효과가 있는지)

⏱️ 시간 관련
├── 시술 소요 시간 (평균)
├── 효능효과 지속기간
├── 다운타임 (회복기간)
└── 다음 시술 시기 (최소 시술 간격)

⚠️ 부작용 정보
├── 흔히 예상되는 부작용 (붓기, 멍 등)
├── 드물게 나타나는 부작용
└── 위험한 부작용 (⚠️ 고지 필요 - 슬쩍이라도 반드시 안내)

📝 주의사항
├── 시술 전 주의사항
│   (금식 여부, 약물 중단, 음주 제한 등)
│
├── 시술 중 주의사항
│   ├── 통증 정도 (1~10점)
│   ├── 통증 종류 (찌릿/따끔/뜨거움/둔한 통증/당기는 느낌 등)
│   ├── 통증 지속 정도
│   ├── 선택 가능한 마취 종류 (크림마취/국소마취/수면마취 등)
│   ├── 마취 적용 시 통증 감소 정도
│   ├── 마취 지속 시간
│   ├── 마취 시 감각 (무감각/둔한 느낌/차가운 느낌 등)
│   ├── 마취 시 나타날 수 있는 부작용
│   └── 마취 전/후 주의사항
│
└── 시술 후 주의사항
    (세안, 운동, 사우나, 음주 등 제한 사항)

🔗 연계 시술 정보
├── 같이 받으면 좋은 시술 (크로스셀링/업셀링 용도)
├── 같이 받으면 안 되는 시술
└── 시술 순서 주의사항
    (예: 보톡스 후 같은 부위 열 치료 → 보톡스 효과 감소 가능)

📊 트렌드 & 선호도 (자동 분석)
├── 시술 유행 정도 (나라별 트렌드)
│   → SNS/커뮤니티 자동 분석
│   → 채팅 문의량 분석
│   → 나라별로 다르게 측정
│
└── 고객 선호도 (자동 측정)
    → CRM 만족도 평가 기반
    → 채팅 문의 빈도 기반
    → 예약 전환률 기반
```

#### 13-2. 기본값 vs 커스터마이징

<!--
  [기획 의도]
  기본값(교과서 기준)을 우리가 잘 세팅해놓는 것이 매우 중요하다.
  클리닉 입장에서 처음부터 모든 걸 입력하기는 부담스럽다.
  기본값이 잘 되어있으면 클리닉은 "자기만 다른 것"만 수정하면 된다.
  의사의 철학, 클리닉 방침, 경험에 따라 다른 부분만 덮어쓰기.
-->

```
우선순위:

1순위: 클리닉이 직접 수정한 값 (덮어쓰기)
       → "우리 클리닉은 보톡스 다운타임 5일로 안내"

2순위: 교과서 기본값 (우리가 세팅)
       → "보톡스 다운타임 기본값: 3일"

→ 클리닉이 수정하지 않은 항목은 교과서 기본값이 자동 적용
→ 클리닉이 수정한 항목은 클리닉 값이 우선
→ 대시보드에서 "기본값"과 "수정됨"을 시각적으로 구분 표시
```

#### 13-3. 정보 접근 권한 (고객에게 보일 것 / 안 보일 것)

<!--
  [기획 의도 - 매우 중요]
  프로시져 허브에는 비즈니스 데이터(재료대, 난이도, 세일즈 점수 등)와
  고객 안내 데이터(효능, 부작용, 주의사항 등)가 함께 있다.
  AI가 고객에게 답변할 때 비즈니스 데이터는 절대 노출하면 안 된다.
  고객이 "이 시술 원가가 얼마예요?"라고 물어도 답하면 안 된다.
-->

```
✅ 고객에게 보이는 정보 (AI 답변에 사용)
├── 시술 설명, 효능효과
├── 수가 (정가, 이벤트가)
├── 시술 시간
├── 부작용 정보 (흔한 + 드문 위험 부작용)
├── 다운타임
├── 시술 전/중/후 주의사항
├── 통증 관련 정보
├── 마취 관련 정보
├── 효능 지속기간
├── 시술 간격
├── 같이 받으면 좋은 시술 (추천으로만, 세일즈 의도 비노출)
└── 시술 후 주의사항

❌ 고객에게 절대 안 보이는 정보 (내부용)
├── 재료비 (원가)
├── 마진, 마진율
├── 시술 난이도 점수
├── 세일즈 퍼포먼스 점수
├── 클리닉 선호 등급
├── 분당 마진
├── 할인율 (49% 경고 등 내부 로직)
└── 상담 퍼포먼스 점수

→ AI 프롬프트에 "위 내부 정보는 어떤 경우에도
   고객에게 노출하지 않는다"를 명시적으로 지시
→ 고객이 원가, 마진 등을 물어보면 정중하게 회피
```

### Part B. 수가(가격) 관리

<!--
  [기획 의도]
  수가 관리는 프로시져 허브의 일부.
  클리닉마다 가격이 다르고, 이벤트가가 다르고, 할인율이 다르다.
  특히 할인율 49% 초과는 의료법 위반이므로 시스템에서 자동 경고해야 한다.

#### 13-4. 수가 입력 항목 (시술당)

```
💰 수가 정보
├── 정가 (원)
├── 이벤트가 (원)
├── 할인율 (%) → 자동 계산: (정가-이벤트가) ÷ 정가 × 100
│   └── ⚠️ 49% 초과 시 → 🔴 빨간색 경고
│       "의료법 위반 가능: 할인율이 49%를 초과합니다"
├── 이벤트 기간 (시작일 ~ 종료일)
├── 패키지 가격 (세트 상품인 경우)
└── 통화별 가격 (JPY, USD, CNY 등 → 자동 환산 or 직접 입력)
```

#### 13-5. 수가 입력 방식

<!--
  [기획 의도]
  엑셀 폼 기반으로 일괄 입력/수정이 가능해야 한다.
  시술 정보 + 수가 + 비즈니스 데이터를 하나의 엑셀에서 관리.
  기존 양식이 있는 클리닉은 폼 컨버터로 변환, 이미지만 있으면 OCR.
  단, 정확성은 클리닉 책임 (면책 필수).
-->

```
방법 1: 엑셀 폼 직접 입력 (기본)
  [📥 프로시져 허브 엑셀 폼 다운로드]
  → 시술 정보 + 수가 + 비즈니스 데이터 통합 양식
  → 입력 후 업로드
  [📤 엑셀 업로드]

방법 2: 기존 양식 변환 (폼 컨버터)
  클리닉 자체 가격표/시술 목록 업로드
  → 폼 컨버터가 우리 양식으로 자동 변환
  → ⚠️ 변환 결과 검토 필수 (클리닉 책임)

방법 3: 이벤트 이미지 OCR
  이벤트 포스터/이미지 업로드 (JPG, PNG)
  → OCR로 시술명, 가격, 기간 자동 추출
  → 우리 양식에 자동 채움
  → ⚠️ OCR 결과 검토 필수 (클리닉 책임)

⚠️ 면책 고지 (모든 방식 공통):
  "변환/OCR 결과는 참고용이며,
   최종 데이터의 정확성은 클리닉 측에서
   반드시 검토 후 확인해주셔야 합니다.
   데이터 오류로 인한 책임은 클리닉에 있습니다."
  → 확인 체크박스 + [확인 완료] 클릭 필수
  → 검토 없이 바로 적용 방지
```

### Part C. 세일즈 퍼포먼스 점수

<!--
  [기획 의도]
  "어떤 시술이 클리닉에 가장 이익인가"를 점수화한 것.
  AI는 이 점수가 높은 시술을 자연스럽게 우선 추천/유도한다.

  고려 요소:
  1. 분당 마진 = (수가 - 재료원가) ÷ 시술시간
     → 시간 대비 수익 효율
  2. 난이도 (5점 만점, 낮을수록 좋음)
     → 난이도 낮은 시술이 클리닉 로딩을 덜 걸리게 함
     → 숙련된 의사는 수가 적고 인건비가 비쌈
  3. 클리닉 선호 등급 (1~3등급)
     → 클리닉의 전략적 판단 반영

  인건비를 직접 넣지 않는 이유:
  변동값이고, 클리닉마다 달라서 정확한 수치를 받기 어려움.
  대신 "난이도"로 간접 반영 (난이도 높음 = 비싼 의사 필요 = 숨은 비용).
-->

#### 13-6. 비즈니스 데이터 입력 항목 (시술당)

```
💰 비즈니스 데이터 (내부용, 고객 비노출)
├── 재료비/소모품비 (클리닉 실제 사입가)
│   → 대량구매 클리닉일수록 낮음 → 점수 올라감
│
├── 시술 난이도 점수 (1~5점)
│   1점: 매우 쉬움 (인턴급 가능)
│   2점: 쉬움 (일반의 가능)
│   3점: 보통 (경력의 필요)
│   4점: 어려움 (전문의 필요)
│   5점: 매우 어려움 (숙련 전문의 한정)
│   → 낮을수록 세일즈 퍼포먼스 점수에 가산
│
├── 시술 시간 (평균, 분 단위)
│
└── 클리닉 선호 등급
    1등급 ⭐: 파는 걸 추천하는 시술 (가산점 최대)
              → 클리닉이 전략적으로 미는 시술
    2등급 🔵: 팔면 좋은 시술 (가산점 보통)
              → 기본 추천 대상
    3등급 ⚪: 팔고 싶지 않은 시술 (가산점 없음 or 감점)
              → 수익성 낮거나, 리소스 대비 비효율적이거나,
                내부 사정으로 줄이고 싶은 시술
```

```

  분당 마진 점수 (40점)
  + 난이도 점수 (30점)
  + 클리닉 선호 점수 (30점)


① 분당 마진 점수 (40점 만점)
   분당 마진 = (수가 - 재료원가) ÷ 시술시간(분)
   → 전체 시술 중 상대 순위로 점수화 (상위 시술 = 40점에 가까움)

② 난이도 점수 (30점 만점)
   1점(매우 쉬움) → 30점
   2점(쉬움)      → 24점
   3점(보통)      → 18점
   4점(어려움)    → 12점
   5점(매우 어려움) → 6점
   → 난이도 낮을수록 높은 점수 (핵심!)

③ 클리닉 선호 점수 (30점 만점)
   1등급 ⭐ → 30점
   2등급 🔵 → 15점
   3등급 ⚪ → 0점
```

### Part D. 상담 퍼포먼스 점수

<!--
  [기획 의도]
  세일즈 퍼포먼스 점수는 "어떤 시술을 밀어야 하는가"의 지표.
  상담 퍼포먼스 점수는 "AI가 실제로 얼마나 잘 팔고 있는가"의 지표.

  세일즈 퍼포먼스가 높은 시술을 AI가 추천했을 때,
  실제로 예약/결제까지 이어졌는지를 측정하는 것.
  즉, AI의 "실전 성적표".

  QR 결제를 통해 모든 결제가 시스템을 통과하므로
  예약 전환률, 결제 전환률을 정확하게 팔로우업할 수 있다.

  KPI: 상담 퍼포먼스 점수 90점 이상 (100점 만점)
-->

#### 13-9. 상담 퍼포먼스 점수 공식

```
상담 퍼포먼스 점수 (100점 만점) =

  세일즈 믹스 점수 (40점)
  + 예약 전환률 점수 (30점)
  + 결제 전환률 점수 (30점)


① 세일즈 믹스 점수 (40점 만점)
   "AI가 세일즈 퍼포먼스 점수 높은 시술을 얼마나 많이 팔았는가"

   = Σ(판매 시술의 세일즈 퍼포먼스 점수 × 판매 건수) ÷ 총 판매 건수
   → 전체 판매 중 고점수 시술의 비중이 높을수록 높은 점수

② 예약 전환률 점수 (30점 만점)
   "상담 → 예약으로 얼마나 이어졌는가"

   예약 전환률 = 예약 건수 ÷ 상담 건수 × 100%
   → 전환률 구간별 점수:
     90%+ → 30점
     80%+ → 25점
     70%+ → 20점
     60%+ → 15점
     50%+ → 10점
     50% 미만 → 5점

③ 결제 전환률 점수 (30점 만점)
   "예약 → 실제 결제(QR)까지 이어졌는가"

   결제 전환률 = 결제 완료 건수 ÷ 예약 건수 × 100%
   → QR 결제 시스템이므로 정확한 팔로우업 가능
   → 전환률 구간별 점수:
     95%+ → 30점
     90%+ → 25점
     85%+ → 20점
     80%+ → 15점
     70%+ → 10점
     70% 미만 → 5점

KPI 목표: 상담 퍼포먼스 점수 90점 이상
```

### Part E. AI의 세일즈 활용

#### 13-12. 추천/유도 시나리오

<!--
  [기획 의도]
  AI가 노골적으로 "이 시술이 마진 좋으니 이거 하세요"라고 하면 안 된다.
  자연스러운 상담 흐름 속에서 세일즈 퍼포먼스 점수 높은 시술을
  슬쩍 우선 노출하는 것이 핵심.
-->

```
1. 열린 질문 → 점수 높은 시술 우선 추천
   고객: "주름 없애고 싶은데 뭐가 좋아요?"
   AI: "보톡스가 가장 인기 있어요! 3분이면 끝나고
        자연스럽게 펴져요😊 필러도 좋은 선택이에요~"
        (보톡스 92점 > 필러 78점 순으로 안내)

2. 업셀링 (추가 판매 유도)
   고객: 보톡스 예약 확정 후
   AI: "이마 보톡스 하실 때 미간도 같이 하시면
        더 자연스러워요! 세트로 하면 할인도 있어요✨"

3. 크로스셀링 (연계 시술 활용)
   → 프로시져 허브의 "같이 받으면 좋은 시술" 정보 활용
   고객: 보톡스 완료 후
   AI: "보톡스 효과 좋으시죠?😊
        다음에 피부 레이저도 같이 받으시면
        피부 톤까지 확 좋아져요~"

4. 고가 시술 문의 시 → 대안 자연스럽게 제시
   고객: "코성형 하고 싶어요"
   AI: "코성형 좋은 선택이에요!
        먼저 코 필러로 라인을 잡아보시는 것도 방법이에요.
        필러는 부담 없이 시작할 수 있어요😊"
        (필러 78점 > 코성형 42점)

5. 시술 순서 안내
   → 프로시져 허브의 "시술 순서 주의사항" 활용
   고객: "보톡스랑 레이저 같이 하고 싶어요"
   AI: "보톡스 먼저 하시고, 레이저는 2주 후에 하시는 게
        효과가 더 좋아요! 순서대로 예약 잡아드릴까요?😊"
```

#### 13-14. 메트릭 설계 원칙

<!--
  [기획 의도]
  점수의 가중치와 구간을 정하는 것이 매우 중요하다.
  잘못 설계하면 AI가 엉뚱한 시술만 추천하게 된다.

  원칙:
  1. 분당 마진, 난이도, 클리닉 선호 3가지를 균형 있게 반영
  2. 클리닉 선호 등급에 30%를 준 이유: 클리닉의 전략적 의지를 존중
     (아무리 수치적으로 좋아도 클리닉이 팔기 싫다면 밀면 안 됨)
  3. 상담 퍼포먼스는 실전 데이터 기반이므로 시간이 지날수록 정확해짐
  4. 메트릭은 정기적으로 리뷰하고 조정해야 함
     (시장 변화, 클리닉 피드백에 따라)
-->

```
메트릭 설계 시 주의사항:

1. 가중치는 조정 가능해야 함
   → 클리닉마다 "분당 마진을 더 중시" or "선호 등급을 더 중시" 등
     우선순위가 다를 수 있음
   → 기본값을 제공하되, 고급 설정에서 조정 가능

2. 점수 구간은 데이터 기반으로 보정
   → 초기: 이론적 구간 설정
   → 운영 후: 실제 데이터 분포에 맞게 구간 재조정
   → 예: 예약 전환률 90%가 현실적으로 불가능하면 구간 하향

3. 이상치 처리
   → 초고가 시술(수천만원)이 점수를 왜곡하지 않도록
   → 분당 마진 점수를 상대 순위(백분위)로 계산

4. 시즌별 변동 반영
   → 여름에 인기 있는 시술, 겨울에 인기 있는 시술이 다름
   → 트렌드 자동 분석이 이를 반영
```

## 14. 세무/법적 구조

<!--
  [기획 의도]
  외국인환자 유치업자가 의료비를 직접 수취하면 의료법 위반 소지가 있다.
  Stripe Connect(자동 분배)도 검토했으나,
  클리닉 통장에 전액이 안 들어오면 매출과 입금액이 안 맞아서
  세무당국에 소명해야 하는 번거로움이 생긴다.

  결론: 업계 관행대로 "클리닉이 전액 수취 → 월말 수수료 정산"이
  세무적으로 가장 깔끔하고 안전하다.
-->

### 14-1. 결제 흐름

```
고객 → 결제 → 클리닉이 전액 수취 (매출 전액 잡힘)
                    │
               월말 정산
                    │
                    ▼
         유치업자에게 수수료 입금
         ├── 수수료: 10%
         ├── 부가세: 별도
         ├── 세금계산서: 유치업자가 발행
         └── 클리닉이 입금

클리닉 장부:
  매출: 50만원 (전액)
  비용: -5만원 (수수료)
  이익: 45만원

✅ 통장 입금액 = 매출액 일치 → 세무상 완벽
```

### 14-2. 필수 법적 요건

```
1. 외국인환자 유치업자 보건복지부 등록
2. 클리닉과 수수료 계약서 (수수료율, 정산 주기 명시)
3. 의료법 전문 변호사 검토 (결제 흐름 구조)
4. 이용약관 / 개인정보처리방침 (결제 정보 처리)
5. AI 상담 관련 고지 의무
```
